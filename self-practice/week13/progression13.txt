- ถ้าไม่มี try...catch เมื่อเกิด error หนึ่งครั้ง ฟังก์ชันจะหยุดทำงานทันทีและ error จะเด้งขึ้นไปจนทำให้หน้าเว็บหรือระบบส่วนอื่นทำงานผิดพลาดตามไปด้วย
- การติดต่อกับ backend เป็นแบบ asynchronous เพราะต้องรอการตอบกลับจาก server ซึ่งใช้เวลาไม่แน่นอน จึงทำให้เมื่อจะทำ CRUD ต้องใช้ async function เพราะต้องรอข้อมูลจาก backend
  const res = await fetch(url); //คำสั่งนี้จะหยุดอยู่ตรงนี้จนกว่าจะได้รับข้อมูลจาก backend จากนั้นจึงทำงานต่อได้
- หลังจาก fetch แล้ว ต้องมีการเช็คว่าผลลัพธ์ที่ได้เป็นตามที่ต้องการมั้ย ถ้าไม่ใช่ -> throw error เพื่อให้ไปจัดการใน catch

-- GET --
- ใช้สำหรับ ดึงข้อมูล จาก backend
- ต้องเป็น async function เพื่อรอผลลัพธ์จาก server
- ใช้ try...catch เพื่อกัน error ทำให้โปรแกรมไม่ล่ม
- fetch(url) ก็เป็น GET อยู่แล้ว ไม่ต้องใส่ method
- ต้องเช็คว่า status = 200 ไม่งั้นให้ throw error
- ใช้ res.json() เพื่อเอาข้อมูลที่ backend ส่งกลับมา

-- POST --
- ฟังก์ชันที่สร้างไว้ต้องรับ 2 param
    - URL ที่จะส่งข้อมูลไป
    - ข้อมูลใหม่ในรูปแบบ JSON obj
- ตอนเรียก fetch ต้องส่งข้อมูลเสริม
    - method: "POST" เพื่อบอก server ว่าต้องการเพิ่มข้อมูล
    - headers เพื่อระบุชนิดข้อมูลที่ส่ง เช่น "Content-Type": "application/json"
    - body ต้องใช้ JSON.stringify() ก่อน เพราะต้องส่งเป็นข้อความไม่ใช่ obj
- การตอบกลับที่ถูกต้องจาก backend สำหรับการสร้างข้อมูลใหม่คือ
    - status = 201 (Created)
- ถ้าไม่ได้สถานะนี้ -> ต้อง throw error เพื่อให้เข้าไปอยู่ในส่วน catch
- เมื่อผ่านเงื่อนไขทุกอย่างแล้ว ควร return ข้อมูลที่ backend ส่งกลับมา เช่น obj ที่สร้างใหม่

-- PUT --
- PUT ใช้เมื่อเราต้องการอัปเดตข้อมูลที่มีอยู่ โดยคล้าย POST แต่มีความหมายต่างกันคือ ต้องมีข้อมูลเดิมอยู่ก่อนถึงจะอัปเดตได้
- ต้องส่งข้อมูลที่editแล้วกลับไปยัง server
- โครงสร้าง fetch ต้องคล้าย POST คือมี method, headers, body
- สถานะที่บอกว่าupdateสำเร็จคือ 200 OK
- ถ้า server ตอบกลับสถานะอื่น ให้ถือว่าการ update fail และต้อง throw error

-- DELETE --
- ฟังก์ชันควรรับ 2 ตัวเหมือน POST และ PUT
    - URL
    - id ของข้อมูลที่ต้องการลบ
- ในการลบข้อมูล จะต้องส่ง id ต่อท้ายใน URL -> fetch(`${url}/${id}`)
- การเช็ค status สามารถใช้ res.ok ได้เลย
- ถ้า true = 200–299 -> ถือว่าสำเร็จ
- ถ้า false = ถือว่า fail
- เมื่อลบเสร็จ ควรคืนค่า id ที่ถูกลบเพื่อให้โปรแกรมรู้ว่าข้อมูลไหนถูกเอาออก
